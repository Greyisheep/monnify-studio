## Defects found and fixed

### QA-001 — Ajo “Add New Member” opened an unrelated workflow picker

- Severity: High
- Area: Business dashboard → Saving Group (Ajo)
- Reproduction: Open the Ajo dashboard and select **Add New Member** in the
  panel header.
- Before: The global “What do you want to do?” workflow-template dialog opened.
- Expected: Start adding a member to the current Ajo rotation.
- Root cause: The header CTA reused the dashboard-level `onNew` callback, whose
  responsibility is creating a new workflow.
- Fix: Exposed an imperative `addMember()` action from `AjoPanel` and connected
  the header CTA to it. The action appends a blank roster row, scrolls the roster
  into view, and focuses the new Name field. It is disabled until an Ajo artifact
  exists.
- Verification: Re-tested in Chromium. One click produced a new roster row with
  focus in `Name 1`; no template dialog opened.

### QA-002 — “View all activites” was misspelled and opened Notifications

- Severity: Medium
- Area: Business dashboard → Activity
- Reproduction: Select **View all activites**.
- Before: A notification dialog opened, even though the Activity card describes
  recent payments and payouts. The label also misspelled “activities”.
- Expected: Move to the complete recent-transaction view.
- Fix: Corrected the label to **View all activities** and connected it to the
  Recent transaction section with smooth scrolling.
- Verification: TypeScript validation passed; the updated control was exercised
  after hot reload and no browser runtime errors were recorded.

### QA-003 — Toolbar falsely labelled a sandbox run as “no real money”

- Severity: High
- Area: Workflow canvas toolbar
- Reproduction: Leave the execution adapter in its default **Sandbox** state and
  inspect or select the toolbar run action.
- Before: The toolbar said **Practice run (no real money)** while the same screen
  showed **Sandbox** and submitted `adapter: monnify`. In the credential-free QA
  environment this produced an HTTP 422; with credentials it would make Monnify
  sandbox requests despite the no-real-money promise.
- Expected: The run label must agree with the selected adapter.
- Fix: Made the toolbar adapter-aware. It now says **Run in Monnify sandbox** for
  the Monnify adapter and reserves **Practice run (no real money)** for the mock
  adapter.
- Verification: Confirmed both labels by toggling the adapter in Chromium.
